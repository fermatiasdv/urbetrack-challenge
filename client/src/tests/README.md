# tests

Tests globales del proyecto (ver [docs/specs/architecture.md](../../../docs/specs/architecture.md), sección "Estrategia de testing"). Los tests unitarios/de componentes viven co-localizados junto a la implementación dentro de cada feature; acá sólo van los que no tienen un único archivo de origen al lado del cual vivir.

- `e2e/` — pruebas end-to-end
- `integration/` — pruebas de integración entre features
- `fixtures/` — datos de prueba compartidos
